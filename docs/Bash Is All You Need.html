My quiet rebellion against over-engineering

The $Bash shell is still the fastest, most deterministic tool you own

In an era obsessed with autonomous agents and thousand-line orchestration frameworks, Bash remains the most reliable way to engineer modern workflows. This is the case for putting intelligence at the edges and keeping the core pipeline rigid.

— Gregory Kennedy: ML Engineer

01Philosophy

Three properties make Bash uniquely suited to sit at the core of non-deterministic probabilistic systems.

01 / Determinism

Exact execution

Unlike non-deterministic probabilistic agents that can hallucinate logic, Bash executes exactly what is written. Exit codes provide binary clarity: success (0) or failure (>0).

02 / Composition

The ultimate glue

Bash isn't meant to write neural networks — it connects the tools that do. Piping curl to jq to python beats a 500-line wrapper for robustness.

03 / Adaptability

Change in seconds

When APIs shift or dependencies break, a shell script is adapted with sed or awk. Modifying a heavy abstraction layer takes hours.

user@ai-workstation:~

— The backdrop

The Machine Learning reality

The glamour of artificial intelligence lies in model architecture and training, but the reality of building these systems is overwhelmingly data-centric. Before a GPU processes a single tensor, terabytes of unstructured text, logs, and images must be fetched, cleaned, filtered, and formatted. This is where high-level abstractions fail and bare-metal scripting shines.

03The Agent Problem

Why deterministic scripts beat autonomous agents for the core loop.

⚠ Agent — non-deterministic & probabilistic

The autonomy trap

Asking an AI agent to decide how to execute a deployment introduces probabilistic failure. If context shifts slightly, the agent picks the wrong tool or loops trying to fix its own errors.

  • Reliability ≈ 45%
  • Heavy abstraction overhead
  • Non-reviewable decision path
✓ Bash — deterministic

Intelligence at the edges

Use the LLM to write a strict, reviewable Bash script once. Humans verify it. Bash executes it deterministically forever. The core pipeline stays rigid.

  • Reliability ≈ 98%
  • Minimal overhead
  • Diff-able, auditable, reproducible

04Fast Facts

A few foundational facts worth internalizing.

History

Born in 1989

Written by Brian Fox for the GNU Project as a free replacement for the Bourne shell (sh).

"Bourne Again Shell" is a pun on the original Bourne shell built by Stephen Bourne at Bell Labs.
Ubiquity

It's Everywhere

Default login shell on most Linux distros; was the macOS default for over a decade.

macOS switched to Zsh in Catalina, but Bash is still bundled and dominates containerized environments.
Feature

The pipe |

The vertical bar hands stdout of one command directly to stdin of the next — the foundation of composition.

ls -l | grep "html" | wc -l
Danger

rm -rf /

With great power comes the ability to delete your entire operating system instantly.

rm -rf / — do not run. Removes files recursively and forcefully from the root.
